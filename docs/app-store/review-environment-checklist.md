# App Review 审核环境清单

## 目标

让审核员无需加入个人 Tailnet、无需等待临时二维码，也能完整验证 Mimi Remote，同时把真实开发环境和账号风险降到最低。

## 方案

- 使用独立测试 Mac、测试用户或可随时重建的隔离环境。
- 只配置一个无敏感信息的 `Mimi Review Sample` 仓库。
- 使用独立 Codex 测试账号与独立 `agentd` Token。
- 通过稳定的公网 HTTPS Endpoint 提供审核访问；不要公开明文 HTTP 端口。
- 审核 Token 在整个审核窗口内有效且可重复连接，审核结束后立即撤销。

## 实现

提交前逐项确认：

- [ ] 后端版本与待提交 App 兼容，`agentd status` 和 `agentd doctor` 通过。
- [ ] Endpoint 使用有效 HTTPS 证书，未暴露个人 Tailnet 地址。
- [ ] Token 仅能访问审核样例目录，不复用个人设备 Token。
- [ ] 样例仓库无真实密钥、个人数据、客户代码和付费资产。
- [ ] 预置至少一个可读取会话、一个干净 Diff 示例和一个可安全审批的小修改。
- [ ] 手动输入 Endpoint 与 Token 可以从全新安装状态连接。
- [ ] 相机、麦克风和语音权限全部拒绝后，核心审核路径仍可完成。
- [ ] 中英文切换无需重启，并且连接页、会话页、设置页均无明显未翻译文本。
- [ ] 审核期间关闭自动休眠、自动重启和会中断连接的维护任务。
- [ ] 使用外部网络完成一次端到端验证，不能只在同一局域网测试。
- [ ] App Review Notes 已填入真实 Endpoint、Token、可用时间和联系手机。
- [ ] 设定审核结束后的 Token 撤销、测试账号退出和环境销毁负责人。

## 风险与优化

- 不要把真实审核 Token 或 Endpoint 提交到 Git；只填写到 App Store Connect 的 App Review Information。
- 正常用户的二维码仍应保持短期、单次使用，不要为了审核削弱生产配对安全。
- 如果 HTTPS 入口出现异常，应先恢复同一审核环境，不要临时切到个人 Mac 或放宽目录/命令权限。
